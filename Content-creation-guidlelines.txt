Content Contribution Guidelines for questions.json

1. Introduction
This document provides the official guidelines for adding new questions to the questions.json data file for the Quiz App. To ensure data consistency, application stability, and a high-quality user experience, it is mandatory to follow this structure precisely.

The goal is to create a rich, well-organized, and easily filterable question bank. Adherence to these guidelines is critical for the filtering and data-parsing logic of the application to function correctly.

2. General JSON Structure
The questions.json file is an array of question objects. Each object represents a single question and must follow the structure below.

High-Level Structure:

{
  "id": Number,
  "sourceInfo": { ... },
  "classification": { ... },
  "tags": [ ... ],
  "properties": { ... },
  "question": "String",
  "question_hi": "String",
  "options": [ ... ],
  "options_hi": [ ... ],
  "correct": "String",
  "explanation": { ... }
}


3. Detailed Field Explanations
id (Required)
Type: Number (Integer)

Description: A unique identifier for the question. It must be sequential. Before adding a new question, check the id of the last question in the file and use the next integer.

Example: 72

sourceInfo (Required)
An object containing metadata about the question's origin.

examName:

Type: String

Description: The name of the exam. Must be one of the predefined values from the list below.

Example: "CGL PRE"

examYear:

Type: Number

Description: The year the exam was conducted.

Example: 2024

examDateShift:

Type: String

Description: Additional information like the date and shift.

Example: "Shift 1" or "Various Shifts"

classification (Required)
An object for the hierarchical categorization of the question. All values must be chosen from the predefined lists provided in Section 4.

subject:

Type: String

Example: "Polity"

topic:

Type: String

Example: "Fundamental Rights & Duties"

subTopic:

Type: String

Example: "Fundamental Rights (Art 12-35)"

tags (Recommended)
Type: Array of Strings

Description: Specific keywords that help in fine-grained searching. These can be more specific than the sub-topic. Use PascalCase for multi-word tags.

Example: ["Article 19", "FreedomOfSpeech", "Constitution"]

properties (Required)
An object containing question attributes.

difficulty:

Type: String

Description: The difficulty level. Must be one of the predefined values.

Example: "Medium"

questionType:

Type: String

Description: The type of question. Must be one of the predefined values.

Example: "MCQ"

question & question_hi (Required)
Type: String

Description: The full text of the question in English (question) and Hindi (question_hi). Include the question number (e.g., Q.72)) at the beginning of the string.

Example (question): "Q.72) Which article of the Indian Constitution guarantees the 'Right to Freedom of Speech and Expression'?"
